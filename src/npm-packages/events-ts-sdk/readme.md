How to build sdk node package:

1. run the events service, so it's hosted at http://localhost:5121 (run it in http mode to avoid certificates troubles) or change the config (orval.config.js)
1. run `orval --config .\orval.config.js `
2. Before publishing set your github personal token in the env variable: GITHUB_NODE_AUTH_TOKEN
2. `cd /src`
4. `npm version patch`
3. `npm run build` 
5. `npm publish`


Can be automated by running bash or powershell scripts: publish.sh or publish.ps1
(Generated by LLM)

Examples:\
`./publish.ps1 -VersionType minor`\
`./publish.sh --version-type patch`

There are three options for the `-VersionType` parameter: `major`, `minor`, and `patch`.

It allows flexibility:
* If you're on a development branch, you might want to bump **minor** (1.0.0 → 1.1.0).
* If you're doing a breaking change, bump **major** (1.x.x → 2.0.0).
* Most of the time, you bump **patch** (1.0.5 → 1.0.6).
